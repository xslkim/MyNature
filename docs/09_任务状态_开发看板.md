# MyNature 任务状态开发看板

> 用途：自动化开发过程中实时记录每个任务的执行状态。
> 更新方式：AI Agent 每完成/阻塞/验证一个任务时自动追加记录；人也可手动编辑。
> 关联文档：`08_任务拆分与执行路线.md`

---

## 状态定义

| 状态 | 含义 | 谁更新 |
|---|---|---|
| `待开始` | 任务已定义，未开始执行 | — |
| `进行中` | AI Agent 或人正在执行 | Agent / 人 |
| `待验证` | 代码/资产已产出，等待验证（编译/运行/对账） | Agent |
| `完成` | 验证通过，可进入下游任务 | Agent / 人 |
| `阻塞` | 因外部依赖/环境问题/需求变更无法继续 | Agent / 人 |
| `跳过` | 经决策不需要执行（如备选方案被采纳） | 人 |

## 自动化更新约定

AI Agent 执行完一个任务后，按以下模板追加日志并更新总表：

```
1. 在「执行日志」追加一行：
   [YYYY-MM-DD HH:MM] {任务ID} 待开始 → {新状态} | {操作摘要} | {验证结果}

2. 在「任务状态总表」找到对应行，更新：
   - 状态 → 新状态
   - 开始时间 / 完成时间（如适用）
   - commit/tag（如已提交）
   - 验证结果
   - 阻塞原因（如状态为阻塞）

3. 如任务进入「完成」状态，检查下游依赖任务的前置是否全部完成，
   若完成则在日志中标注「{下游任务ID} 前置就绪，可开始」。
```
