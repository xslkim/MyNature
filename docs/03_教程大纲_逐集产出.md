> **项目目的**：通过从零复刻 IL3DN 插画式自然场景（Demo + 冬季松林），让有 **1 年经验的 Unity 开发者**跟着视频一步一步做，最终独立完美复刻整个项目。
>
> **本文件**：教学主线。每一集 = 一集视频，有独立的可演示成果、知识点、产出物。
>
> **配套文档**：工程阶段见 `02_复刻计划与里程碑.md`；视频制作格式见 `04_视频制作规范_AutoVideo.md`；源项目资料见 `01_源项目调研报告.md`。

## 观众画像与教学设计

**观众画像**：约 1 年 Unity 经验。
- ✅ 已会：编辑器基本操作、C# 基础、Prefab/材质/场景概念、MonoBehaviour 生命周期
- ⚠️ 没做过：手写 HLSL、URP 自定义渲染、Blender 建模、程序化生成、Editor 批量装配脚本
- 🎯 需求：**每一步都能在屏幕上跟着做**，做完有明确成果

**设计原则**：
1. **每集可独立演示**：看完一集，屏幕上有一个明确的"东西变了/多了"。
2. **知识点单一聚焦**：一集只深入 1-2 个新概念；新概念第一次出现时必讲原理。
3. **难度递增 + 支柱交替**：HLSL / Blender / Python / C# / Editor 脚本交替，避免连续多集同质。
4. **失败可恢复**：每集结束 commit + tag，任何一集的代码都能独立跑通到该集状态。
5. **先原理后代码**：每个 shader/系统先用 1-2 分钟讲"为什么这么做"，再动手写。
6. **视觉素材录屏为主**：视频画面以 Unity 编辑器/运行录屏为主体（详见 `04` 文档）。

# 第三部分：建模入门（E04）

## E04 — Blender MCP 建第一棵树：松树低模

**主题**：Blender MCP 工作流入门 + 顶点色编码约定（全项目的"暗号"）

**知识点**
- MCP 原理：socket server（9876）→ `execute_blender_code` 跑 bpy
- 程序化建模思路：圆柱树干 + 多层锥形树冠，参数化函数
- **顶点色编码约定**：`.g` = 叶子/上层权重（影响雪与风）、`.a` = 风权重——E05/E07 的 shader 就靠它（⚠️ 开涂色前先看 05 规格的"逐 shader 通道表"最终确认）
- FBX 导出面板参数：**Forward = `-Z`、Up = `Y`** + Apply Scalings = FBX Units Scale（Blender→Unity 轴向原理讲清楚；批量导出后由脚本固化，见 06 规格）+ Unity Model Import 设置
- 最小贴图包概念：为什么先出 5 张核心贴图把画面立住（完整生成体系 E10 再讲）

**演示动作**
1. 写 ~60 行 bpy：清空场景 → 圆柱 → 锥形树冠 → 逐顶点涂色（g=1, a=0.8 渐变）
2. MCP 执行，Blender 视口看模型
3. 导出 `Pine_01.fbx` → Unity 导入
4. **最小贴图包**：跑 `gen_minimal.py` 出 5 张核心贴图（bark/leaf/grass/pine/windnoise）→ 松树换上 bark/pine 贴图，画面立刻有插画感雏形
5. 用顶点色可视化验证（Scene 视图 debug 模式）

**产出物**：`Pine_01.blend` + `Pine_01.fbx` + 最小贴图包 5 张 + 顶点色验证

**录屏素材**：MCP 调用过程、Blender 视口建模、Unity 导入、贴图换装前后、顶点色可视化

**commit 标签**：`e04-first-pine-blender`

# 第四部分：风系统（E05）

## E05 — 风的原理：全局变量契约 + Branch 着色器

**主题**：整套视觉系统的核心机制——C# 每帧推全局变量，shader 读变量摆顶点

**知识点**
- `Shader.SetGlobalVector/Float/Texture` 机制：C# ↔ shader 的唯一耦合点
- 全局变量契约全表：`WindDirection`、`NoiseTextureFloat`、`WindStrenghtFloat`（拼写错误保留！）等
- Shader keyword：`[Toggle(_WIND_ON)]` + `#pragma multi_compile`
- 世界空间 noise 平移 → 顶点位移的 HLSL 写法；为什么要抽 `Includes/IL3DN_Wind.hlsl`
- URP PBR 骨架：`UniversalFragmentPBR`、`GetMainLight`（URP 10 → 17.5 API 差异对照）

**演示动作**
1. 写 `Includes/IL3DN_Noise.hlsl`（snoise）+ `IL3DN_Wind.hlsl`（位移函数）
2. 写 `IL3DN_Branch.shader`（源 1076 行，聚焦核心 ~300 行讲透）
3. 写 `Scripts/Runtime/IL3DN_Wind.cs`（全局风驱动器 MonoBehaviour）
4. E04 的松树换 Branch 材质 → 场景里摆动起来
5. Inspector 拖 `WindStrenght` 滑杆：风强弱实时可见

**产出物**：2 个 include + `Branch.shader` + `Wind.cs`，松树随风摆动

**录屏素材**：shader 关键段讲解（配合代码高亮）、滑杆调风、有风/无风对比

# 第六部分：植被着色器（E07）

## E07 — Grass / Leaf / Pine + 雪系统

**主题**：三种植被 shader 的差异与组合，引入一键换季的雪系统

**知识点**
- Grass：UV rotation wiggle（与世界空间 noise 不同的第二种摆动）
- Leaf：+ alpha cutoff 全局调节（`AlphaCutoffFloat`）
- Pine：风 + wiggle + 雪三合一（最复杂植被 shader）
- 雪累积 mask：`saturate(pow(WorldNormal.y, k)) * SnowXxxFloat`——按世界法线 Y 积雪
- keyword 组合：`_WIND_ON` / `_WIGGLE_ON` / `_SNOW_ON`
- `IL3DN_Snow.cs`：6 类表面独立雪量 global

**演示动作**
1. 写 `Includes/IL3DN_Snow.hlsl`
2. 写 3 个 shader（Grass 1207 / Leaf 1256 / Pine 1229 行，按"差异点"讲而非逐行）
3. 写 `Snow.cs`
4. 草/花/松全部换装 → 风动效全员就位
5. **高光时刻**：Snow 开关从 0 拖到 1，整个测试场景一键入冬

**产出物**：3 shader + `Snow.hlsl` + `Snow.cs` + 5 材质，风/雪双系统可见

**录屏素材**：草 wiggle 特写、松针摆动、雪量滑杆 0→1 渐变过程（本集封面素材）
