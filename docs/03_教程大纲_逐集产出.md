# MyNature 教程大纲（逐集产出）

> **项目目的**：通过从零复刻 IL3DN 插画式自然场景，学习 Unity 6 + URP 17 + 手写 HLSL + Blender MCP + Python 程序化 + Unity CLI 自动化的完整工作流。
>
> **本文件**：教学版主线。每一集 = 一集视频，有独立的可演示成果、知识点、产出物。
>
> **配套工程视角**：见 `02_复刻计划与里程碑.md`。
> **源项目资料**：见 `01_源项目调研报告.md`。

---

## 设计原则

1. **每集可独立演示**：观众看完一集，屏幕上能看到一个明确的"东西变了/多了"。
2. **知识点单一聚焦**：一集只深入 1-2 个新概念，不混杂。
3. **难度递增**：E01 从零开始，E16 收尾整套场景。技术栈分 4 个支柱交替穿插，避免连续多集同质。
4. **失败可恢复**：每集结束 commit 一次，任何一集的代码都能独立跑通到该集状态。
5. **真实工作流**：不是"我教你点鼠标"，而是"我作为开发者如何用 AI agent 高效产出"——展示 ZCode + Unity CLI + Blender MCP 的真实协作。

---

# 第六部分：着色器——草、叶、松（E07）

## E07 — 风系统 2 + 雪系统：Grass / Leaf / Pine 三着色器

**主题**：把风系统扩展到草（UV wiggle）+ 叶（摆动）+ 松（组合），并引入雪系统

**知识点**
- Grass 着色器：UV rotation wiggle（不同于世界空间 noise）+ 顶点色 `.g`/`.a` 双通道
- Leaf 着色器：在 Grass 基础上加 alpha cutoff 全局调节（`AlphaCutoffFloat`）
- Pine 着色器：风 + wiggle + 雪三合一（最复杂的植被着色器，1228 行）
- 雪累积 mask：`saturate(pow(WorldNormal.y, k)) * SnowXxxFloat`
- Shader keyword：`_SNOW_ON`、`_WIGGLE_ON`、`_WIND_ON`

**演示动作**
1. 写 `IL3DN_Grass.shader`（1206 行）、`IL3DN_Leaf.shader`（1255 行）、`IL3DN_Pine.shader`（1228 行）

3. 给 E06 的草/花加 Grass shader，给 E04/E05 的松加 Pine shader
4. 屏幕上：草 wiggle、松针组合摆动；勾 `Snow=true` 一键变雪季

**产出物**

# 附录 B：单集时长与节奏建议

| 集类型 | 建议时长 | 节奏 |
|---|---|---|
| 配置/工具链（E01, E02） | 15-20 分钟 | 慢，演示每一步 |
| 着色器集（E03, E05, E07, E09, E12） | 25-35 分钟 | 中，重点讲原理再写代码 |
| Blender 建模集（E04, E06, E08） | 20-30 分钟 | 快，bpy 脚本可跳读 |
| Python 贴图（E10） | 25 分钟 | 中，噪声原理稍展开 |
| C# 脚本（E11） | 30 分钟 | 中，FPS 控制器原理 |
| CLI 自动化（E13, E14, E15） | 25-35 分钟 | 慢，这是差异化亮点 |
| 收尾（E16） | 20 分钟 | 总结性 |

**总时长估计**：约 7-9 小时视频内容。
