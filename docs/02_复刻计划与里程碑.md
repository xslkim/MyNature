# MyNature 复刻计划与里程碑

> 目标：在 `G:\unity_demo\MyNature` 从零复刻 IL3DN 的 **Demo 场景**（高保真）。
> 源项目调研：见 `01_源项目调研报告.md`。

---

## 1. 关键技术决策（已与用户确认）

| 维度 | 决策 | 说明 |
|---|---|---|
| Unity 版本 | **Unity 6 LTS**（6000.x） | 升级到最新 LTS，使用 URP 17 |
| 复刻范围 | **仅 Demo 场景**，高保真 | 植被/树/石种类齐全、季节系统完整 |
| 着色器 | **手写 HLSL**（照搬逻辑，适配 URP 17） | 不用 Shader Graph |
| 贴图 | **程序化生成**（Python/PIL） | 色卡、噪声、渐变、法线全代码生成 |
| 3D 模型 | **Blender MCP 生成** | 通过 Blender 的 MCP 服务端驱动建模 |
| 音频 | **复用源项目**（66 wav + mixer） | 唯一允许复制源资源 |
| 自动化装配 | **Unity CLI + com.unity.pipeline** | 2025 新出的官方 CLI；`unity command eval` 在运行中的 Editor 里跑 C#，用于批量创建材质/prefab、刷地形、截屏验证 |

### 工具链总览（本项目的"四大支柱"）

```
┌──────────────────────────────────────────────────────────────┐
│  1. Unity 6 LTS + URP 17          ← 渲染与运行时              │
│  2. 手写 HLSL (.shader/.hlsl)     ← 视觉风格                  │
│  3. Blender MCP (execute_blender_code 跑 bpy)  ← 3D 建模     │
│  4. Python/PIL                    ← 程序化贴图                │
│  + Unity CLI (unity command eval) ← 自动化 Asset 配置/装配    │
└──────────────────────────────────────────────────────────────┘
```

每集教程通常围绕其中 1-2 个支柱展开，保证单集知识点聚焦。

### 决策带来的张力与对策

- **Unity 6 + 手写 HLSL**：原着色器用 URP 10 API，Unity 6 是 URP 17，`UniversalFragmentPBR` 等 API 签名/包含路径有变化 → **不是纯照搬，而是"逻辑照搬 + API 迁移"**。迁移工作量集中在 4-6 个核心着色器（Branch/Pine/Leaf/Grass/Terrain/Water）。
- **Built-in RP 雾效失效**：`OnRenderImage` / PPv2 在 Unity 6 URP 不工作 → 改写为 **URP Full Screen Pass Renderer Feature**。
- **Blender MCP 环境缺失**：当前系统未检测到 Blender 与 MCP 服务端 → **阶段 0 必须先搭环境**。

## 2. 总体技术架构（MyNature）

```
MyNature/
├── Assets/
│   ├── Audio/                      # ← 复制自源项目（66 wav + mixer）
│   ├── MyNature/
│   │   ├── Art/
│   │   │   ├── Materials/          # ~50 mat（重建后引用新 shader + 新贴图）
│   │   │   ├── Meshes/             # Blender MCP 导出的 FBX
│   │   │   ├── Models/             # Blender 源文件 .blend（备查）
│   │   │   └── Textures/           # Python 生成（PNG，统一不用 PSD）
│   │   ├── Prefabs/                # 树/草/石/花/环境/系统 prefab
│   │   ├── Scenes/                 # Demo 场景 + 子 profile
│   │   ├── Scripts/
│   │   │   ├── Runtime/            # 11 个 MonoBehaviour（适配 Unity 6）
│   │   │   └── Editor/             # 5 个 CustomEditor
│   │   ├── Settings/               # URP Asset / Renderer / Volume Profile
│   │   └── Shaders/                # 15 个手写 HLSL（URP 17 适配后）
│   │       └── Includes/           # 新增：公共 .hlsl（风/雪/噪声全局变量封装）
├── Packages/manifest.json          # Unity 6 版依赖
├── ProjectSettings/                # Unity 6 默认 + URP 配置
└── docs/                           # 本文档系列
```

### 架构变化（相对源项目）

1. **新增 `Shaders/Includes/`**：把 snoise、风计算、雪计算抽成共享 `.hlsl`，避免每个 shader 重复 inline（原项目是复制粘贴）。
2. **新增 `Art/Models/.blend`**：保留 Blender 源文件，方便后续修改。
3. **PSD → PNG**：程序化生成直接出 PNG（避免 PSD 工具链依赖）。
4. **雾效重构成 Renderer Feature**：替代原 Built-in `OnRenderImage`。
5. **保留全局变量命名**：`WindStrenghtFloat`（拼写错误也保留）等，确保 C# 与 shader 解耦契约不变。

### 阶段 0：环境与项目骨架（预估 1 天）

**目标**：Unity 6 项目可打开、URP 可用、Unity CLI/pipeline 可驱动、Blender MCP 可调用、Python 贴图工具链可用。

- [ ] 0.1 用 Unity Hub 创建 Unity 6 LTS URP 项目到 `G:\unity_demo\MyNature`（3D URP 模板）
- [ ] 0.2 安装 **Unity CLI**（Windows PowerShell beta channel：`$env:UNITY_CLI_CHANNEL='beta'; irm https://public-cdn.cloud.unity3d.com/hub/prod/cli/install.ps1 | iex`）

- [ ] 0.4 验证 `unity command eval "UnityEngine.Debug.Log(\"hello\")"` 能在运行中的 Editor 里执行（这是后续所有自动化的基础）
- [ ] 0.5 安装 Blender 4.2 LTS + [blender-mcp](https://github.com/ahujasid/blender-mcp) 插件，验证 MCP `get_scene_info` 可返回

- [ ] 0.9 配置 URP Asset（质量档、阴影、HDR、抗锯齿）

**验收**：四条链路全通——Unity 能跑、`unity command eval` 有响应、Blender MCP 能建一个立方体、Python 能输出测试 PNG。

### 阶段 4：3D 模型（Blender MCP）（预估 3-4 天）⭐ 工作量大

**目标**：~40 个 FBX 由 Blender MCP 生成，顶点色编码遵守 `.g`=叶子权重、`.a`=风权重。

按优先级分组（先做 Demo 场景高频资产）：

- [ ] 4.1 **草簇 ×3**（`Plant_Grass_01/02/03`）—— Demo 中 ~8000 株，最高频
- [ ] 4.2 **枯叶/枯枝贴花 ×8**（`Dead_Leaf_*`、`Dead_Branch_*`、含 _Decal 变体）—— ~5300 个
- [ ] 4.3 **岩石 ×3**（`Rock_Boulder_01/02/03`）—— ~700 个
- [ ] 4.4 **花 ×3**（`Plant_Flower_01/02/03`）—— ~700 个
- [ ] 4.5 **睡莲 ×1**（`Plant_WaterLily_01`）—— 610 个
- [ ] 4.6 **松树 ×3**（`Pine_01/02/03`，每个含完整 + `_OneMesh`）—— 含 billboard 卡
- [ ] 4.7 **桦树 ×3**（`Birch_01/02/03`）
- [ ] 4.8 **灌木 ×3**（`Shrub_Round_01/02/03`）
- [ ] 4.9 **山毛榉 ×3**（`Beech_01/02/03`，可复用 Birch 网格改色）
- [ ] 4.10 **环境小物**：`GroundPatch`、`Human_scale`（标尺）、`Water_Plane`、`LakeFog`、`Props_Bridge`、`Cloud`（云可用 plane+alpha 替代建模）

> **建模策略**：
> - 草/叶/花/睡莲等 → **面片 + alpha**（平面几何 + cutout 材质，简单高效）
> - 树干/树枝 → **低模圆柱细分** + 顶点色编码
> - 树冠 → **十字卡 + alpha**（billboard），不建实体叶模型
> - 岩石 → **细分立方 + 噪声扰动**
> - 每个模型导出 `.blend` 源 + `.fbx`（Y 轴向前、Z 向上适配 Unity）

**验收**：每个 FBX 导入 Unity 后有正确的 Mesh、材质槽、顶点色。

## 5. 风险与备选方案

| 风险 | 概率 | 备选 |
|---|---|---|
| Blender MCP 在 Windows 环境不稳 | 中 | 退化为手写 Blender Python 脚本批量建模；或手工建关键资产 |
| URP 17 着色器 API 迁移卡壳 | 中 | 退化为 Unity 内置 Simple Lit + 颜色 tint，牺牲风/雪特效 |
| 1.6 万实例性能不足 | 低 | Terrain Detail + Tree 替代纯 Prefab；启用 GPU Instancing |
| 程序化贴图风格不像 | 中 | 阶段 8 调色 + 在线找 CC0 手绘贴图补 |
| Unity 6 LTS 某些 Asset Store 资源不可用 | 低 | 全部用代码/Blender 自建，不依赖外部资源 |
