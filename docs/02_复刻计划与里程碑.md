## 4A. 端到端自动化架构（2026-07-22 调研定稿）

> 回答一个问题：**不用 Unity CLI，是不是就必须手动操作编辑器？——不是。**
> 自动化的核心从来不是某个 CLI，而是"业务逻辑写在 Editor 脚本里 + 触发方式可插拔"。

### 4A.1 三层架构

```
┌─ L2 编排层：AI Agent（Trae）或 PowerShell 脚本，按里程碑顺序串起所有步骤
│
├─ L1 触发层（四选一或可叠加，与 L0 解耦）：
│   ① Unity.exe -batchmode -executeMethod MyNature.Builder.Xxx -quit
│      ← 保底主力。Unity 自带、零依赖、无头运行、CI 级稳定
│   ② Unity CLI + com.unity.pipeline（unity command run/eval）
│      ← 2026-07-20 官方发布，beta 实验包；驱动运行中的 Editor，免逐次域重载
│   ③ 官方 Unity MCP Server（com.unity.ai.assistant）
│      ← 需 Unity Cloud 连接 + Unity AI 订阅；MCP 工具化驱动 Editor
│   ④ 社区 unity-mcp（WebSocket 插件，免费）
│      ← 与 Blender MCP 同形态，AI Agent 直连运行中的 Editor
│
└─ L0 业务层：Editor 脚本（C# static 方法）
    MyNatureBuilder_Materials / _Prefabs / _Terrain / _Screenshots / _Recorder
    ← 所有"干活"的代码只写在这里，四种触发器调的都是同一套方法
```

### 4A.2 各环节自动化率评估

| 环节 | 自动化方式 | 自动化率 |
|---|---|---|
| 目录骨架 / 音频复制 / git | PowerShell 脚本 | 100% |
| Shader / C# 代码编写 | AI Agent 直接写文件 + batchmode 编译验证 | ~100% |
| 贴图生成 | Python 脚本 | 100% |
| 3D 模型 | Blender MCP（已就绪） | ~95%（个别造型需人工微调） |
| 材质 / Prefab / 变体 | L0 Builder 脚本 + batchmode 触发 | 100% |
| 地形创建 / 1.6 万撒点 | L0 Builder 脚本 + batchmode 触发 | 100% |
| 场景装配参数还原 | L0 Builder 脚本（参数全部来自调研报告） | ~95% |
| **视觉校色 / 调优** | 脚本自动截图 → 人眼/Agent 视觉对比 → 调参迭代 | **~50%，必须保留人工判断** |
| 静态截图素材 | RenderTexture / ScreenCapture（batchmode 带 GPU，不加 `-nographics`） | 100% |
| 漫游/效果类录屏 | Unity Recorder API（RecorderController）+ 代码驱动相机路径 → 直接出 1080p30 mp4 | ~90% |
| **编辑器 UI 操作类录屏**（教学演示用） | Editor 脚本自动执行操作 + OBS 录屏 | **~30%，需要人录制** |
| 视频构建 | AutoVideo 构建 Agent（BUILD.md 流程） | 100% |

### 4A.3 结论

- **工程复刻**：除"视觉校色最终判断"外可全链路无人值守。一批次跑通 ≈ 一条 PowerShell 总控脚本顺序调 N 次 batchmode。
- **视频素材**：静态截图与漫游录屏可脚本化自动产出；只有"编辑器 UI 操作演示"类镜头需要人工录（约占录屏素材 20-30%）。
- **Unity CLI 的定位**：2026-07-20 官方刚发布（Unite Seoul），`com.unity.pipeline` 仍是 beta 实验包。优势是驱动**运行中的** Editor、免去每次 batchmode 的启动与域重载开销，适合高频交互式调试。**作为可选加速，不作为依赖**；E01 时可花 10 分钟试装，好用就留，不好用不影响任何计划。
- **Unity MCP（官方/社区）**：与 Blender MCP 同形态，适合"Agent 边做边看"的交互式装配；同样可选，安装与否不改变 L0 脚本设计。

> 教学侧说明：E14 按 L0（Builder 脚本）+ ①（batchmode）为主线教学，②③④ 作为"进阶彩蛋"在集尾提一句，避免观众环境差异导致跟做失败。

## 6. 风险与备选方案

| 风险 | 概率 | 备选 |
|---|---|---|
| URP 17.5 着色器 API 迁移卡壳 | 中 | 单 shader 退化为 URP Simple Lit + tint，先保流程后补特效 |
| Blender MCP 长脚本超时（180s） | 中 | 拆小脚本分批执行；退化手写 bpy 文件本地跑 |
| 1.6 万实例性能不足 | 低 | Terrain Detail/Tree 替代纯 Prefab；GPU Instancing；降密度 50% |
| 程序化贴图风格不像 | 中 | 阶段 9 调色；CC0 手绘贴图补关键几张 |
| Water 5 pass 太复杂 | 中 | 砍 ShadowCaster/DepthOnly 只留 Forward（牺牲水下深度） |
| 旧 Input Manager 在 Both 模式下行为差异 | 低 | 切回仅 Input Manager（模板 Input System 包可留不用） |
| 冬季场景无源文件可对照 | 中 | 以冬季 Profile + PineForest 布局 + 官方宣传图/视频为参考基准 |
